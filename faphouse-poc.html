F-01 — faphouse.com Expires April 15, 2026 — Registrar Name.com — Auto-Renew Unconfirmed — Split DNS/Registration Management
faphouse.com  |  rdap.verisign.com  |  Domain Expiry Risk
RDAP QUERY — rdap.verisign.com/com/v1/domain/faphouse.com
GET https://rdap.verisign.com/com/v1/domain/faphouse.com HTTP/1.1 200 OK ldhName: faphouse.com status: client transfer prohibited registration: 2021-04-15T19:56:10Z expiration: 2026-04-15T19:56:10Z ← EXPIRES IN ~12 DAYS at 19:56 UTC registrar: Name.com, Inc. (IANA ID 625) ← NOT Cloudflare Registrar nameservers: jake.ns.cloudflare.com / nia.ns.cloudflare.com Auto-renew status: NOT visible via RDAP — internal registrar billing flag only. SPLIT MANAGEMENT: Registered at Name.com. DNS managed at Cloudflare. Renewal notices go to Name.com billing contact — not Cloudflare dashboard.
ImpactDomain lapse path: active until 19:56 UTC Apr 15 → auto-renew grace ~30 days (domain functions, recoverable) → redemption (DNS goes dark, $80–150 fee) → pending delete 5 days (unrecoverable) → drop (caught within seconds). A platform with 1.37M premium subscribers and active payment flows cannot safely lapse. The Name.com/Cloudflare split means renewal notices route to Name.com billing — which may not be the active admin.
RemediationLog into Name.com (not Cloudflare). Verify billing card. Enable auto-renew. Manually renew before April 15. Do not rely on Cloudflare notifications — registration is at Name.com.

F-19 — 7,915 Performer Records Returned Unauthenticated Including HLS Stream URLs for Paid Private, Group, and Virtual Private Shows
vr.faphouselive.com/api/vr/v2/models  |  Performer Directory API  |  Unauthenticated Data Exposure
GET /api/vr/v2/models — zero credentials
GET https://vr.faphouselive.com/api/vr/v2/models HTTP/1.1 200 OK Content-Type: application/json; charset=utf-8 Access-Control-Allow-Origin: * Response size: 8,241,692 bytes (8.2 MB) performers_total: 7,915 show_breakdown: publicChat: 7,453 ← free tier privateChat: 234 ← paid $/min — HLS URL included groupShow: 123 ← ticket-gated — HLS URL included virtualPrivate: 112 ← premium rate — HLS URL included fields_per_performer: username, country, gender, isLive, showType, hlsPlaylist, ← direct delivery URL — no auth required to use privateRate, spyRate, groupRate, ticketRate, broadcastSettings, previewImages[], biography, languages, tags
ImpactEvery live performer including those in paid sessions is returned with their direct HLS delivery URL and per-minute rates. This is the enumeration surface for F-20 — the full stream pipeline access bypass.
RemediationRequire auth on /api/vr/v2/models. Unauthenticated callers: return only publicChat, omit hlsPlaylist entirely, omit per-minute rates.

F-20 — HLS Delivery Chain Zero-Auth Across All Tiers — Private, Group, Virtual Private Shows Accessible Without Payment
manifest.naiadsystems.com · edge-hls.doppiocdn.com · media-hls.doppiocdn.com  |  Three-Layer HLS Pipeline  |  Payment Bypass
THREE-LAYER PIPELINE — all layers HTTP 200, no authentication
LAYER 1 — manifest.naiadsystems.com GET https://manifest.naiadsystems.com/live/s:{stream_id}/index.m3u8 HTTP/1.1 200 OK CORS: * #EXTM3U #EXT-X-MOUFLON:{token} ← present but NOT enforced at CDN #EXT-X-STREAM-INF:BANDWIDTH=500000,RESOLUTION=426x240 https://edge-hls.doppiocdn.com/hls/{id}/master/{id}_240p.m3u8 #EXT-X-STREAM-INF:BANDWIDTH=1400000,RESOLUTION=1280x720 https://edge-hls.doppiocdn.com/hls/{id}/master/{id}_720p.m3u8 LAYER 2 — edge-hls.doppiocdn.com GET https://edge-hls.doppiocdn.com/hls/{id}/master/{id}_720p.m3u8 HTTP/1.1 200 OK CORS: * LAYER 3 — media-hls.doppiocdn.com (segments) GET https://media-hls.doppiocdn.com/hls/{id}/{seg}.m4s HTTP/1.1 200 OK Content-Type: video/mp4 TIER COVERAGE CONFIRMED: PUBLIC (publicChat): 200 OK expected PRIVATE (privateChat): 200 OK paywall bypassed — viewers paying $/min GROUP (groupShow): 200 OK paywall bypassed — ticket required VIRTUAL PRIVATE: 200 OK paywall bypassed — premium rate tier
ImpactEvery paid session type is fully accessible without credentials. The MOUFLON token field is present in the manifest but unenforced at the Bottlenose CDN delivery layer. The stream UUID does not rotate on classification change — access obtained during a public phase persists through a transition to paid. Revenue loss scales with platform traffic.
RemediationToken-signed manifest and segment URLs at the Bottlenose CDN layer. Tokens must be short-lived (≤300s), session-bound, tier-validated server-side. Enforce the MOUFLON field at CDN origin. Remove hlsPlaylist from unauthenticated API responses. Rotate stream UUID on public→paid transitions.

F-02 — Subscription Architecture + Internal Tier Configuration Fully Exposed Unauthenticated (1.37M VOD, tierConfigurationIds, Payment Stack)
faphouse.com/api/premium/get-packages  |  Subscription API  |  Business Logic Exposure
GET /api/premium/get-packages — no credentials
GET https://faphouse.com/api/premium/get-packages HTTP/1.1 200 OK 9,549 bytes "tierId": 23 "tierTitle": "16.99 tier" "premiumVideosCount": 1,371,008 ← full catalogue size "tierConfigurationIds": [260, 262, 265] ← P3 entitlement targets "accessLevel strings": ["premium_lite", "ultra"] "methods": ["RocketGate","Epoch","CoinGate","PayGarden","CentroBill"]
ImpactComplete internal subscription architecture exposed: internal IDs used in entitlement decisions, access level strings used in authorization checks, full payment processor stack, exact catalogue size. tierConfigurationIds (260, 262, 265) are P3 targets — substituting a higher-tier ID in an authenticated request may upgrade access without payment.
RemediationStrip internal IDs, access level strings, payment processor names, and catalogue counts from unauthenticated responses. Return only public-facing price/duration/tier name.

F-03 — faphouselive.com: No Content Security Policy — XSS Fully Unmitigated
faphouselive.com  |  HTTP Headers  |  Missing Security Controls
GET https://faphouselive.com/ 200 OK Content-Security-Policy: ABSENT X-Frame-Options: ABSENT X-Content-Type-Options: ABSENT All standard security headers absent. Any XSS is uncontained browser-wide.
ImpactXSS on any surface enables session hijack, subscriber payment exfiltration, and affiliate commission manipulation. No browser-level mitigation exists.
RemediationDeploy CSP report-only then enforce. Add X-Frame-Options: DENY, X-Content-Type-Options: nosniff.
